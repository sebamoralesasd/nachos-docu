#define FUSE_USE_VERSION   26

A FUSE client the Nachos file system.

FUSE (Filesystem in Userspace) is a mechanism for integrating custom file systems from userspace in POSIX operating systems. This program allows the user to mount Nachos' file system in a directory and then access it using all the standard tools (e.g. commands like ls and cat, or graphical file managers).

For now, only read capabilities are implemented in the FUSE client. This means that it is yet not possible to create a new file or modify an existing one through the standard tools.

Another limitation is that the DISK file, which contains the whole simulated disk content, must be available in the same directory where the FUSE client is executed. It is recommended to set up a symbolic link to the original in the filesys directory. If you launch the client with make mount, the link gets created automatically.
